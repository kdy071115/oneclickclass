- `PASS`: 현재 목 환경에서 기대 결과를 실제 상호작용 또는 코드 경로로 확인

- `BLOCKED`: 실제 API, SSE, 외부 앱 또는 시각 기준 이미지가 없어 완전 검증 불가

## 4. 반응형 결과

### Desktop 1440x900

- 20개 핵심 라우트에서 `scrollWidth - clientWidth = 0`
- 빈 페이지 없음
- 클래스 관리 탭, 수료증, 수강 화면 주요 콘텐츠 표시

### Tablet 1024x768

- 11개 핵심 라우트에서 가로 넘침 없음
- 사이드바 폭 264px 고정: 접힘 요구사항 실패
- 본문 폭 760px로 축소되어 일부 화면의 정보 밀도가 높음

### Mobile 390x844

- 14개 핵심 라우트에서 가로 넘침 없음
- `.phone` 컨테이너 폭 390px 유지
- 데스크톱 전용 수강 화면은 숨겨지고 모바일 수강 화면 표시

## 5. 접근성 점검

- 핵심 8개 화면의 보이는 이미지에는 누락된 `alt`가 발견되지 않음
- 대시보드·클래스·상세·출석·신청자·수료증·수강 화면의 보이는 컨트롤은 이름 존재
- 설정 화면에서 이름 없는 보이는 컨트롤 5개 발견: 알림/다크 모드 토글에 `aria-label` 필요
- 팝오버 Escape 닫기는 동작하지만 포커스 복원 없음
- `dialog` 컴포넌트는 네이티브 dialog를 사용하지만 명시적인 포커스 순환 검증/구현 없음

## 6. 우선 수정 권장 순서

1. `P0`: 수료증 실제 PDF 생성, 시험 실제 채점, 학습 재생·진도 처리
2. `P0`: 수정 화면 기존 클래스 데이터 로딩과 저장 결과 영속화
3. `P1`: 상단 검색, 신청자/출석 CSV, 회차 선택, 설문 결과 화면
4. `P1`: 알림 읽음 배지 동기화, 토글 접근성 이름, 팝오버 포커스 복원
5. `P1`: 1024px 태블릿 사이드바 접힘
6. `P2`: 다크 모드 실제 테마, 시각 회귀 기준 이미지 구축

## 7. 테스트 한계

- 목 데이터는 생성·수정 결과를 메모리 목록에 영구 반영하지 않음
- PG 결제, 환불 웹훅, 실제 이메일/문자, 실제 SSE는 서버 환경 필요
- PDF는 현재 브라우저 인쇄 호출이라 파일 내용 자동 검증 불가
- 색 대비와 픽셀 회귀는 전용 도구/기준 이미지 추가 후 재검증 필요
