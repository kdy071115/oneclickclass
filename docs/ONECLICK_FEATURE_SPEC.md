# 원클릭 클래스 기능 명세서

> 기준일: 2026-07-23
> 기준 코드: 현재 프론트엔드 구현 및 `docs/oneclick-lx2-api.md`
> 목표: LX2를 회원·과정·차수·수강신청·진도의 기준 시스템으로 사용하고, 원클릭 클래스는 강의 개설과 신청·학습 UX를 제공한다.

## 1. 범위와 상태 기준

| 상태 | 의미 |
|---|---|
| UI | 화면과 사용자 상호작용 구현 여부 |
| 프론트 로직 | 검증·상태 전환·API 호출 구현 여부 |
| Mock API | 로컬 모의 저장과 자동 테스트 연결 여부 |
| 실제 API | LX2·PG·SMS 등 운영 API 연결 여부 |
| 운영 검증 | 권한·동시성·보안·장애 상황 검증 여부 |
| 제외 | 현재 제품 범위에서 구현하지 않음 |

기존 본문의 `구현`은 **UI + 프론트 로직 + Mock API 완료**를 의미한다. 실제 API 또는 운영 검증 완료를 뜻하지 않는다.

상태·결제·좌석·권한의 서버 기준은 [`ONECLICK_STATE_POLICY.md`](./ONECLICK_STATE_POLICY.md)를 우선 적용한다.

### 제외 범위

- SCORM 고급 설정
- 다국어 자막 관리
- 영상 타임라인 퀴즈
- 복잡한 평가 배점식
- 기업·기관별 권한 관리
- 관리자 메뉴 즐겨찾기
- 여러 승인 단계와 결재선

## 2. 핵심 사용자와 식별자

### 사용자

| 사용자 | 목적 | 인증 방식 |
|---|---|---|
| 강의자 | 강의 생성, 공개, 신청자와 운영 데이터 관리 | 일반 로그인 세션 |
| 수강생 | 공유 링크 신청, 결제·승인 확인, 강의 수강 | 신청 후 자동 세션, 새 기기는 휴대전화 인증 |

### 식별자

| 필드 | 역할 | 기준 시스템 |
|---|---|---|
| `courseMasterSeq` | 과정의 공통 원형 | LX2 과정 마스터 |
| `courseActiveSeq` | 실제 모집·운영되는 강의 차수 | LX2 과정 차수 |
| `shareToken` | 공개 신청 페이지 접근용 비예측 토큰 | 원클릭 공개 서비스 |
| `courseApplySeq` | 사용자별 수강신청 | LX2 수강신청 |
| `memberSeq` | 내부 사용자 | LX2 회원 |
| `lessonId` | 프론트에서 사용하는 안정적인 차시 ID | 원클릭 wrapper |
| `organizationSeq` | LCMS 콘텐츠 조직 | LX2 LCMS organization |
| `itemSeq` | 조직 내부 학습 항목 | LX2 LCMS item |
| `activeElementSeq` | 운영 강의에 배치된 차시 | LX2 course active element |
| `contentsSeq` | 원본 학습 콘텐츠 | LX2 LCMS contents |

`shareToken`은 사용자 인증값으로 사용하지 않는다. 신청 완료 후에는 세션 쿠키와 `courseApplySeq` 권한으로 강의실에 접근한다.

## 3. 강의 상태 모델

강의 진행 상태와 모집 상태를 분리한다.

```ts
courseLifecycleStatus: DRAFT | READY | IN_PROGRESS | ENDED | SUSPENDED
recruitmentStatus: PRIVATE | OPEN | CLOSED | FULL
```

현재 프론트의 `CURRICULUM`, `RECRUITING`은 화면 단계 호환값이다. 실제 API 연동 시 `CURRICULUM`은 `DRAFT`의 작성 단계로, `RECRUITING`은 `recruitmentStatus=OPEN`으로 변환하며 단일 운영 상태로 저장하지 않는다.

## 4. 인증과 진입

### AUTH-01 강의자 로그인

- 경로: `/login`
- 입력: 이메일, 비밀번호
- 성공: 세션 저장 후 `/dashboard` 이동
- 실패: 입력 오류 또는 로그인 실패 메시지 표시
- 권한: `/classes/new`, `/settlements`는 강의자만 접근
- 상태: **부분 구현** — 프론트 보호 라우트와 모의 인증 구현, LX2 세션 연동 필요

### AUTH-02 강의자 회원가입

- 경로: `/signup`
- 입력 검증과 역할 선택을 제공한다.
- 상태: **부분 구현** — 실제 회원 생성과 약관 이력 저장 필요

### AUTH-03 수강생 자동 세션

- 공개 링크에서 신청 완료 시 내부 회원과 수강신청을 생성한다.
- 브라우저에는 식별정보가 아닌 HttpOnly 랜덤 세션을 발급한다.
- 상태: **백엔드 필요** — 프론트는 신청 상태 기반 자동 진입 흐름 구현

### AUTH-04 새 기기 이어보기

- 경로: `/learn/:courseActiveSeq`
- 세션이 없으면 휴대전화 번호 → 인증번호 → 기존 수강권 연결 순서로 진행한다.
- 인증번호 불일치·만료 시 진입을 막는다.
- 상태: **부분 구현** — UI와 모의 인증 구현, 실제 SMS·세션 발급 필요

### AUTH-05 비회원 조회

- 기획 결정: 기존 비회원 코드 조회 방식은 제거하고 공유 링크 + 자동 세션 방식으로 통합한다.
- 현재 점검: `/guest` 라우트와 `GuestPage`가 코드에 남아 있다.
- 상태: **기획 충돌 / 제거 필요(P0)**

## 5. 강의자 기능

### TEACHER-01 대시보드

- 경로: `/dashboard`
- 신규 신청, 진행 강의, 결제 대기, 최근 활동을 표시한다.
- 카드와 목록에서 관련 관리 화면으로 이동한다.
- 로딩 중 스켈레톤, 조회 실패 시 재시도 상태가 필요하다.
- 상태: **부분 구현** — 화면·모의 데이터 구현, LX2 집계 API 필요

### TEACHER-02 강의 목록

- 경로: `/classes`
- 상태 필터: 전체, 준비중, 모집중, 진행중, 종료
- 강의 선택 시 `/classes/:courseActiveSeq`로 이동한다.
- 상태·유형·일정·신청 인원·정원을 표시한다.
- 상태: **구현**

### TEACHER-03 강의 생성

- 경로: `/classes/new`
- 순서: 기본 정보 → 진행 방식·장소 → 일정 → 가격·정원 → 신청 정보
- 필수 검증: 제목, 일정, 오프라인·혼합형 장소, 유료 가격
- 생성 결과: `courseMasterSeq`, `courseActiveSeq` 발급 후 비공개 `DRAFT`로 저장
- 다음 화면: `/classes/:courseActiveSeq/curriculum?setup=1`
- 영상·라이브·자료 URL은 기본 정보가 아니라 차시에서 입력한다.
- 상태: **구현**, 실제 LX2 생성 API 필요

### TEACHER-04 강의 수정

- 진입: 상세 화면의 `강의 수정`
- 기존 강의값을 동일 생성 폼에 불러온다.
- 저장 후 동일 `courseActiveSeq` 상세로 복귀한다.
- 구버전 로컬 초안이 정식 강의 정보를 덮어쓰지 않도록 저장 스키마를 구분한다.
- 상태: **구현**

### TEACHER-05 커리큘럼 섹션 관리

- 경로: `/classes/:courseActiveSeq/curriculum`
- 섹션 추가, 이름 수정, 삭제, 위·아래 순서 변경을 제공한다.
- 섹션 삭제 시 포함 차시 수를 확인하고 사용자 확인을 받는다.
- 상태: **구현**, 학습 이력이 있는 섹션 삭제 정책은 백엔드 필요

### TEACHER-06 차시 관리

- 차시 추가, 수정, 삭제, 순서 변경을 제공한다.
- 필드: 제목, 설명, 콘텐츠 유형, 콘텐츠 URL, 예상 시간, 무료 미리보기, 필수 차시, 순차 학습, 공개 여부
- 콘텐츠 유형: `video`, `live`, `document`, `assignment`
- 공개 차시는 유효한 콘텐츠 URL이 반드시 있어야 한다.
- 상태: **구현**

### TEACHER-07 콘텐츠 URL 판별

- YouTube, Vimeo, 일반 영상 파일, 라이브 링크, 문서·과제 링크를 자동 판별한다.
- URL은 `http://` 또는 `https://`로 시작해야 한다.
- 불완전한 YouTube 주소와 잘못된 Vimeo 주소는 저장을 차단한다.
- 상태: **구현**

### TEACHER-08 공개 전 점검

- 경로: `/classes/:courseActiveSeq/preview`
- 점검 항목: 제목, 한 줄 소개, 일정, 장소, 유료 가격, 공개 차시, 차시 콘텐츠 URL
- 실패: 공개 버튼 비활성화, 수정할 화면으로 바로 이동하는 링크 제공
- 성공: 공개 버튼 활성화
- 상태: **구현**

### TEACHER-09 강의 공개와 공유

- 공개 시 `RECRUITING` 상태로 전환하고 `shareToken`을 발급한다.
- 완료 화면에서 링크 복사, 신청 페이지 열기, QR 생성, 공유, 신청자 현황 이동을 제공한다.
- 공개 URL: `/s/:shareToken`
- 상태: **구현**, 운영용 토큰 생성·폐기 정책은 백엔드 필요

### TEACHER-10 강의 상세

- 경로: `/classes/:courseActiveSeq`
- 표시: 상태, 유형, 소개, 신청·정원, 만족도, 완료율, 커리큘럼, 최근 활동, 공유 링크
- 탭: 개요, 신청자, 출석/QR, 설문·시험, 수료증, 설정
- 모든 탭은 동일 `courseActiveSeq` 범위 데이터를 사용한다.
- 상태: **구현**

### TEACHER-11 신청자 관리

- 강의별 경로: `/classes/:courseActiveSeq/applicants`
- 검색, 결제 상태 필터, 상세 이동, CSV 내보내기를 제공한다.

### TEACHER-12 출석과 QR

- QR 발급·새로고침, 만료 시간, 출석·지각·결석 상태를 제공한다.
- 실시간 체크인은 강의 범위 이벤트로 갱신한다.
- 상태: **부분 구현** — 모의 실시간 갱신 구현, LX2 출석 저장 필요

## 6. 신청과 결제 기능

### APPLY-01 공개 신청 페이지 조회

- 경로: `/s/:shareToken`
- 표시: 강의 소개, 강사, 난이도, 일정·장소, 가격, 잔여 좌석, 커리큘럼, 후기
- `shareToken`으로 실제 `courseActiveSeq`를 조회한다.
- 기존 수강권이 있으면 신규 신청 폼 대신 상태 또는 이어보기 CTA를 우선 표시한다.
- 상태: **구현**

### APPLY-02 무료 강의 신청

- 필수 입력: 이름, 휴대전화, 개인정보 동의, 휴대전화 인증
- 선택 입력: 이메일
- 성공 상태: `APPROVED + NOT_REQUIRED + AVAILABLE`
- 공개 차시가 있으면 완료 화면에서 바로 강의실 입장 가능
- 상태: **구현**, 실제 회원 조회·생성과 세션 발급은 백엔드 필요

### APPLY-03 유료 강의 신청

- 무료 신청 필드에 결제·환불 안내 동의를 추가한다.
- 신청·결제·수강권 상태를 각각 `applicationStatus`, `paymentStatus`, `enrollmentStatus`로 반환한다.
- 결제 주문 생성 시 좌석을 15분 임시 점유하고 결제 완료 시 확정한다.
- `APPLY_STATUS::004`, `APPLY_STATUS::002`는 LX2 호환 코드로만 사용한다.
- 상태: **부분 구현** — 결제 대기·완료 UI 구현, 실제 PG 연동 필요

### APPLY-04 승인형 신청

- MVP는 승인 후 결제 순서로 고정한다.
- 신청 후 `applicationStatus=APPLIED` 승인 대기를 표시한다.
- 상태 다시 확인으로 최신 수강신청 상태를 조회한다.
- 상태: **부분 구현** — 단일 승인 상태 UI만 포함, 실제 승인 운영 API 필요

### APPLY-05 신청 완료 화면

- 결제 대기, 승인 대기, 수강 가능, 강의 준비 중 상태를 구분한다.
- 가능한 행동만 주 버튼으로 표시한다.
- 강의 정보 보기로 신청 페이지에 돌아갈 수 있어 페이지에 갇히지 않는다.
- 상태: **구현**

### APPLY-06 모집 제한

- 모집 마감이면 신청 버튼을 비활성화한다.
- 잔여 좌석은 `capacity - confirmedCount - heldCount`로 계산한다.
- 유료 결제 좌석은 15분 임시 점유하고 실패·취소·만료 시 해제한다.
- 중복 신청은 현재 세션의 기존 수강권을 우선 표시한다.
- 상태: **부분 구현** — 서버의 동시 신청 정원 검증과 중복 방지 필요

## 7. 수강생 기능

### LEARN-01 강의실 진입

- 경로: `/learn/:courseActiveSeq`
- 수강 가능 상태와 현재 세션 소유권을 확인한다.
- 수강권이 없으면 휴대전화 인증 화면을 표시한다.
- 공개 차시가 없으면 강의 준비 중 상태와 강의 정보 이동을 제공한다.
- 상태: **구현**, 서버 권한 검증 필요

### LEARN-02 콘텐츠 재생

- 지원: 직접 영상 파일, YouTube, Vimeo, 외부 라이브·문서·과제 링크
- YouTube와 직접 영상은 현재 위치를 저장하고 이어보기를 제공한다.
- Vimeo와 외부 링크는 현재 화면에서 정밀 재생 위치를 수집하지 않는다.
- 상태: **부분 구현** — Vimeo Player API 및 외부 콘텐츠 완료 처리 필요

### LEARN-03 진도 저장과 이어보기

- 재생 중 약 10초 간격, 일시정지, 종료 시 heartbeat를 전송한다.
- 프론트 요청: `lessonId`, `courseApplySeq`, 현재 초, 전체 길이, 재생 여부
- LX2 저장 키: `courseActiveSeq`, `courseApplySeq`, `organizationSeq`, `itemSeq`
- wrapper는 `lessonId`를 신뢰해 직접 저장하지 않고 현재 커리큘럼의 LX2 식별자로 변환한다.
- LX2 `progressMeasure`는 `0~1`, 프론트 `progressPercent`는 `0~100`으로 단위를 분리한다.
- 영상은 90% 이상 시청 또는 종료 이벤트 수신 시 완료로 판정한다.
- `progressPercent`, `completedAt`, `completionReason`을 분리해 저장한다.
- 다시 접속하면 최근 차시와 저장 위치에서 시작한다.
- 이미 완료한 강의도 처음부터 다시 볼 수 있다.
- 상태: **구현**, LX2 LCMS 학습 이력 연결 필요

### LEARN-04 순차 학습

- `sequential=true` 차시는 현재 수강생에게 공개된 직전 학습 대상 차시 완료 전 잠긴다.
- 직전 차시가 선택 차시여도 순차 배열에 포함되면 완료해야 한다.
- 비공개·삭제·접근 불가 차시는 선행 차시 계산에서 제외한다.
- 상태: **구현**, 자동 테스트 완료

### LEARN-05 필수 차시

- 강의자가 차시별 `required`를 설정하고 수강생 데이터에 전달한다.
- `progressRate`와 `requiredCompletionRate`를 분리한다.
- 수료는 필수 차시, 필수 설문, 시험 합격, 출석 기준을 서버에서 조합한다.
- 상태: **부분 구현 / 수료 기준 연결 필요(P1)**

### LEARN-05A 관심 강의

- 수강생의 관심 강의는 LX2 과정 즐겨찾기와 연결한다.
- 관리자 메뉴 즐겨찾기와 같은 기능으로 취급하지 않는다.
- 프론트는 `courseActiveSeq`만 전달하고 `referenceSeq`, 회원 식별자, 등록·삭제 방식은 wrapper가 결정한다.
- 관심 상태 변경 실패 시 낙관적 UI를 원복하고 다시 시도할 수 있어야 한다.
- 상태: **서비스 계약 구현 / 화면 연결 필요(P1)**

### LEARN-06 공지사항

- 미확인 공지 수와 공지 목록을 표시한다.
- 확인 시 읽음 상태와 미확인 수를 함께 갱신한다.
- 상태: **부분 구현** — UI·모의 저장 구현, LX2 게시판 읽음 이력 필요

### LEARN-07 자료실

- 자료 제목, 설명, 유형, 다운로드·열기 동작을 제공한다.
- URL이 없으면 준비 중으로 표시한다.
- 상태: **부분 구현** — UI 구현, 실제 자료 목록·권한 URL 필요

### LEARN-08 설문

- 문항 응답, 필수 응답 검증, 제출 완료 화면을 제공한다.
- 완료한 설문은 중복 참여를 막는다.
- 상태: **부분 구현** — UI·모의 저장 구현, LX2 설문 응답 연동 필요

### LEARN-09 시험

- 문제 풀이, 제출, 서버 채점 결과 형태의 점수·정답 수·합격 여부를 표시한다.
- 문제 조회 응답에는 운영 환경에서 정답을 포함하지 않는다.
- 상태: **부분 구현** — UI·모의 채점 구현, 서버 채점 필요

### LEARN-10 수강 후기

- 별점과 10~500자 후기를 작성·수정·삭제한다.
- 수강신청 한 건당 후기 하나를 유지한다.
- 공개 신청 페이지에는 후기 목록과 평균을 표시한다.
- 상태: **부분 구현** — UI·모의 저장 구현, 소유권 검증·이름 마스킹 필요

### LEARN-11 수료증 조회

- 목록, 상세, 세로형 수료증 표시를 제공한다.
- 상태: **부분 구현** — 화면 구현, 실제 발급 데이터와 PDF 다운로드 필요

## 8. 공통 UX 규칙

- 데스크톱과 모바일은 같은 데이터와 상태를 사용한다.
- 모바일 전용 화면으로 강제 이동하지 않고 현재 뷰포트에 맞춰 반응형으로 렌더링한다.
- 같은 화면에서 동일한 주요 CTA를 중복 노출하지 않는다.
- 로딩, 빈 상태, 오류, 재시도, 비활성 이유를 화면에 표시한다.
- URL에 `undefined`, 사용자 ID, 전화번호, 개인 인증 코드를 노출하지 않는다.
- 각 하위 화면에는 목록·상세·강의 정보로 돌아갈 수 있는 탈출 경로를 제공한다.
- 콘텐츠 URL과 결제·신청 값은 프론트뿐 아니라 서버에서도 다시 검증한다.

## 9. API 연동 요약

상세 요청·응답 형식은 [`oneclick-lx2-api.md`](./oneclick-lx2-api.md)를 기준으로 한다.

| 도메인 | 핵심 API | 프론트 계약·화면 | 실제 백엔드 연동 |
|---|---|---|---|
| 강의 | `/classes`, `/classes/{courseActiveSeq}/detail`, `/publish` | 준비됨 | LX2 래퍼 필요 |
| 커리큘럼 | `/classes/{courseActiveSeq}/curriculum/**` | 준비됨 | 차시·LCMS 매핑 필요 |
| 신청 | `/oneclick/shares/{shareToken}`, `/apply` | 준비됨 | 회원·수강신청·세션 필요 |
| 강의실 | `/oneclick/learn/{courseActiveSeq}/room` | 준비됨 | 권한·목록 조합 필요 |
| 진도 | `/oneclick/learn/{courseActiveSeq}/heartbeat` | 준비됨 | LCMS 이력 저장 필요 |
| 새 기기 | `/verification-codes`, `/continue` | 준비됨 | SMS·세션 필요 |
| 후기 | `/reviews`, `/review` | 준비됨 | 소유권·마스킹 필요 |
| 운영 | 신청자·출석·설문·수료증·설정 API | 일부 준비 | LX2 서비스 연결 필요 |
| 결제·정산 | 결제 주문·완료·환불, 정산 요약·내역 | 일부 준비 | PG 원장 필요 |

`준비됨`은 프론트 화면, 상태 분기, mock 응답 및 호출 인터페이스가 있다는 뜻이다. 운영 API가 구현·검증되었다는 뜻이 아니다.

## 10. 테스트 수용 기준

| 영역 | 현재 자동 검증 |
|---|---|
| 단위·서비스 | 7개 파일, 35개 테스트 통과 |
| 브라우저 E2E | 26개 통과, 기기별 비대상 10개 제외 |
| 강의 생성 | 생성 → 커리큘럼 → 공개 준비 이동 검증 |
| 콘텐츠 | 잘못된 URL 차단, YouTube 자동 판별 검증 |
| 전체 여정 | YouTube 차시 공개 → 신청 → 강의실 입장 검증 |
| 순차 학습 | 이전 차시 완료 전 잠금, 완료 후 해제 검증 |
| 반응형 | 강의자 1024·1280, 수강생 모바일 가로 넘침 검증 |

## 11. 점검 결과와 우선순위

### P0 배포 전 필수

1. `/guest`와 비회원 조회 진입점을 제거하고 공유 링크·자동 세션 흐름으로 통일한다.
2. 강의 진행 상태와 모집 상태를 분리한다.
3. 신청·승인·결제·수강권 상태를 분리하고 취소·거절·환불 회수를 정의한다.
4. 좌석 15분 임시 점유, 만료 해제, 확정 트랜잭션을 구현한다.
5. 최초 신청에서 휴대전화 소유를 확인하고 회원 병합 정책을 적용한다.
6. 결제 콜백을 금액 검증·멱등·원자적 수강권 발급으로 처리한다.
7. 공개 후 강의·커리큘럼 변경 제한과 이력 보존 정책을 적용한다.
8. LX2 JSON 래퍼에서 식별자와 강의 소유권을 검증한다.
9. HttpOnly·Secure·SameSite 세션 쿠키를 적용한다.

### P1 운영 안정화

1. 콘텐츠 유형별 완료 기준과 영상 90% 완료 기준을 적용한다.
2. 필수 완료율과 전체 진도율을 분리해 수료 판정과 연결한다.
3. 순차 학습의 직전 공개 학습 대상 차시 규칙을 적용한다.
4. 무료 미리보기 접근·보안 정책을 확정한다.
5. SMS 인증번호 만료·재전송 제한·실패 횟수를 서버에서 관리한다.
6. 강의·커리큘럼 변경 후 수강생 강의실 캐시 무효화 정책을 적용한다.

### P2 후속 개선

1. 번들 크기 경고를 줄이기 위해 운영·학습 화면 단위 코드 분할을 검토한다.
2. 실제 API 지연·실패·세션 만료 E2E를 추가한다.
3. 접근성 키보드 이동과 스크린리더 회귀 테스트를 추가한다.
